# 이감 모의고사 주문 취합 시스템 — 설치가이드

잇올 독학관리학원(수만휘, 지점 40곳)의 모의고사 주문을 웹으로 접수하고 중앙에서 취합하는 시스템입니다.
학생 개별 주문부터 학원(지점) 승인, 본사 취합까지 한 흐름으로 처리합니다.
설치는 최초 1회, 약 10분이면 끝납니다. 이후 운영은 웹 페이지에서만 합니다.

# 1. 시스템 구성

```
[학생]                     [학원(지점)]                  [본사 관리자]
student.html?b=주문코드    branch.html?k=관리코드         admin.html
학생 개별 주문          →  자기 지점 주문 취합·승인/반려  →  전 지점 취합·확정 → 이감 전달
                              │                              │
[지점 담당자 일괄 주문]        │                              │
index.html ────────────→  구글 스프레드시트  ←──────────────┘
                          (Apps Script 웹 앱 API)
                          시트: 주문 / 지점 / 상품 / 설정
```

| 구성 요소 | 역할 | 비용 |
|---|---|---|
| GitHub Pages | 주문·관리 페이지 게시 (cs-link와 동일 방식) | 무료 |
| 구글 스프레드시트 | 주문 데이터 저장소 (직접 열어 볼 수도 있음) | 무료 |
| Apps Script | 페이지와 시트를 연결하는 API | 무료 |

| 페이지 | 사용자 | 접근 방법 |
|---|---|---|
| student.html | 학생 | 지점별 학생 주문링크 (admin 지점 관리에서 복사해 배포) |
| branch.html | 지점 담당자 | 지점별 학원 관리링크 (지점 담당자에게만 전달) |
| index.html | 지점 담당자 | 일괄 주문이 필요할 때 (링크 하나를 전 지점 공용) |
| admin.html | 본사 관리자 | 관리 토큰 필요 |

주문 상태 흐름: 접수 → (지점) 지점승인 또는 반려 → (본사) 확정 → 결제완료 → 출고. 취소는 본사만 가능.

# 2. 최초 1회 설치 (본사 관리자)

## 2.1 구글 시트 + Apps Script 만들기

1. gugeodat00 계정으로 https://sheets.new 접속, 새 스프레드시트 생성. 이름은 "잇올 주문취합" 등 자유.
2. 상단 메뉴 확장 프로그램 > Apps Script 클릭.
3. 열린 편집기의 기존 코드를 모두 지우고, 이 저장소 `apps-script/code.gs` 파일 내용 전체를 붙여넣기 후 저장(Ctrl+S).
4. 편집기 상단 함수 선택 드롭다운에서 `setup` 선택 후 실행 버튼 클릭.
   - 권한 요청 창이 뜨면 "권한 검토" → 계정 선택 → "고급" → "안전하지 않은 페이지로 이동" → 허용. (본인이 만든 스크립트라 안전합니다.)
5. 실행이 끝나면 스프레드시트에 주문·지점·상품·설정 4개 시트가 생깁니다.
   - "설정" 시트에 관리 토큰이 표시됩니다. 이 값을 복사해 두세요.

## 2.2 웹 앱으로 배포

1. Apps Script 편집기 우측 상단 배포 > 새 배포.
2. 톱니바퀴에서 유형 "웹 앱" 선택.
3. 설정: 실행 계정 = "나", 액세스 권한 = "모든 사용자". 배포 클릭.
4. 발급된 웹 앱 URL(`https://script.google.com/macros/s/..../exec`)을 복사.

## 2.3 페이지와 연결

방법 A (권장) — config.js에 URL 고정:
1. 이 저장소의 `config.js`를 열어 `apiUrl: ""` 안에 웹 앱 URL을 붙여넣고 커밋.
2. 이후 학생·지점·관리 페이지가 모두 자동으로 연결됩니다.

방법 B — 관리 페이지에서만 임시 연결:
1. admin.html 접속 → 설정 탭 → URL과 관리 토큰 입력 → 저장 후 연결 테스트.
2. 이 방법은 해당 브라우저에서만 유효하며, 학생·지점 페이지는 config.js에 URL이 있어야 작동합니다.

관리 토큰은 방법 A/B 어느 쪽이든 admin.html 설정 탭에 1회 입력해야 합니다(브라우저에 저장됨).

## 2.4 초기 데이터 등록

1. admin.html 상품 관리 탭: 상품명·단가 확인 후 수정, 상품 저장.
2. admin.html 지점 관리 탭: 40개 지점명을 한 줄에 하나씩 붙여넣고 일괄 추가 → 지점 저장.
3. 지점 저장이 끝나면 지점마다 학생 주문링크와 학원 관리링크가 자동 생성됩니다. 표에서 복사해 배포하세요.

# 3. 운영 방법

## 3.1 학생 주문 흐름 (기본)

| 단계 | 주체 | 내용 |
|---|---|---|
| 1 | 본사 | admin 지점 관리에서 지점별 학생 주문링크 복사 → 지점에 전달 |
| 2 | 지점 | 학생들에게 주문링크 배포 (카톡 공지 등) |
| 3 | 학생 | 링크 접속 → 이름·연락처·수량 입력 → 제출 (상태: 접수) |
| 4 | 지점 | 학원 관리링크(branch.html) 접속 → 주문 확인 → 승인/반려 (대기 전체 승인 버튼 지원) |
| 5 | 본사 | admin에서 "지점승인 이상 (발주 대상)" 필터로 집계 → 이감 발주 → 확정·결제완료·출고 처리 |

## 3.2 지점 일괄 주문 (보조)

학생 개별 주문 없이 지점이 수량만 모아 주문하는 경우 index.html 링크 하나를 공용으로 사용합니다. 이 주문은 구분이 "지점"으로 표시되며 승인 절차 없이 본사가 바로 확정할 수 있습니다.

## 3.3 본사 취합 관리 (admin.html)

| 탭 | 용도 |
|---|---|
| 주문 취합 | 전체 주문 조회, 기간·지점·구분(학생/지점)·상태 필터, 상태 변경, 중복 의심 자동 표시, CSV 다운로드 |
| 집계 리포트 | 지점 × 상품 수량 피벗, 지점별 금액 정산표, "지점승인 이상" 필터로 발주 수량 확정, 피벗 CSV |
| 지점 관리 | 지점 추가·삭제·사용 여부, 지점별 학생 주문링크·학원 관리링크 복사 |
| 상품 관리 | 상품·단가 추가·수정 (이후 주문부터 적용) |
| 수동 입력 | 카톡·문자·엑셀로 받은 주문을 붙여넣어 표준 주문으로 등록 |

## 3.4 중복·오류 정리 규칙

- 학생 주문: 같은 학생(이름+연락처)이 같은 날 같은 상품을 두 번 이상 주문하면 "중복의심" 표시.
- 지점 일괄 주문: 같은 지점이 같은 날 같은 상품을 두 번 이상 주문하면 "중복의심" 표시.
- 반려·취소된 주문은 수량·금액 집계에서 자동 제외됩니다.

# 4. 주의 사항

- 학원 관리링크(branch.html)는 그 지점의 학생 이름·연락처가 보이므로 지점 담당자 외에는 전달하지 마세요. 학생에게는 학생 주문링크만 배포합니다.
- 관리 토큰이 노출되면 Apps Script 편집기에서 프로젝트 설정 > 스크립트 속성의 ADMIN_TOKEN 값을 바꾸고, 설정 시트와 admin.html에 새 값을 다시 입력하세요.
- 지점을 삭제해도 그 지점의 기존 주문 데이터는 시트에 남습니다. 삭제된 지점의 링크는 더 이상 작동하지 않습니다.
- code.gs를 수정한 경우 배포 > 배포 관리 > 수정(연필) > 새 버전으로 다시 배포해야 반영됩니다.
- 주문 원본은 구글 시트에 그대로 남습니다. 시트를 직접 수정하면 취합 화면과 어긋날 수 있으니, 가급적 관리 페이지에서 처리하세요.
